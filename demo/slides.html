Per-Channel Smart Accounts

Hide merchant identity even at channel-open time

Problem: The escrow requires msg.sender == payee for setMerchantCommitment() and close(). If the merchant uses their real address, the link is visible.

Solution: A factory deploys a fresh ChannelPayee contract per session via CREATE2.

1
Merchant generates a one-time keypair for this session
2
Factory.computePayeeAddress(key) → deterministic address (before deploy)
3
Server tells payer to open channel to that address — not the merchant's real address
4
Factory.deploy(key) — anyone can call, no special permissions
5
Relayers forward merchant-signed instructions (setCommitment, close)

On-chain: payee = 0x8a3f… — a disposable contract, not a merchant

Architecture

Three phases — same vouchers, completely different settlement path

① OPEN + BIND
Payer deposits tokens to escrow
Payee calls setMerchantCommitment
commitment = Poseidon(pubKey, blinding)
Opaque hash — pubKey never on-chain
Required before any settle
② SETTLE
Verify EIP-712 voucher signature
Compute delta from cumulative amount
Mint noteCommitment into Merkle tree
= Poseidon(asset, Δ, mc, ch, rand)

💰 Funds stay in contract
③ REDEEM (ZK)
Submit Groth16 proof
Verifier checks note preimage
+ Merkle membership + nullifier
ALL private inputs stay hidden

🔒 No link to original payer

Trust Model

✅ Trustless
Settlement amounts — contract computes deltas from cumulative vouchers
Recipient assignment — merchantCommitment bound at channel open
Double-spend — nullifier set is on-chain state
Note validity — Groth16 verifier is immutable contract
⚡ Privacy strength scales with
Number of merchants sharing the escrow (anonymity set)
Redemption timing (delayed redemption prevents correlation)
Amount uniformity (micro-payments are harder to distinguish)
